# GUID 重新生成

为选中的 Unity 资源（文件或文件夹）重新生成 GUID，并自动扫描整个 `Assets/` 目录，将所有引用旧 GUID 的地方批量替换为新 GUID，确保引用完整性。

---

## 适用场景

- 将资源从其他项目复制过来后，与现有资源发生 GUID 冲突
- 需要将某个资源作为完全独立的副本使用，不与原始资源共享引用
- 版本控制合并冲突导致 GUID 重复

---

## 使用步骤

1. 在 **Project 窗口**中选中需要处理的资源（支持多选、文件夹）
2. 打开 **XY 编辑器工具集**，在左侧选择 **GUID 重新生成**
3. 工具中间区域会显示当前选中的资源列表，确认无误后选择操作模式
4. 点击 **仅文件** 或 **文件 + 文件夹** 按钮
5. 在弹出的确认对话框中点击 `Yes, please` 开始执行
6. 等待进度条完成，在弹出的结果对话框中查看统计，详细日志见 Console

---

## 操作模式

### 仅文件

- 选中**单个文件**：只处理该文件
- 选中**文件夹**：递归处理文件夹内所有文件，**不修改文件夹自身的 GUID**
- 适合大多数情况

### 文件 + 文件夹

- 在「仅文件」的基础上，同时为文件夹本身也生成新的 GUID
- 当文件夹 GUID 本身也存在冲突时使用

---

## 执行过程

工具分两个阶段运行，底部进度条实时显示进度：

**第一阶段 — 扫描引用**
扫描 `Assets/` 下所有资源，建立「哪些文件引用了选中资源的 GUID」的映射表。

**第二阶段 — 替换 GUID**
- 修改选中资源的 `.meta` 文件，将旧 GUID 替换为新 GUID
- 遍历所有引用文件，将文件内容中的旧 GUID 替换为新 GUID

执行完成后弹出统计对话框，并在 Console 输出详细报告：

```
GUID Regenerator 1.0.4
3 Updated Asset/s
  2 references    Assets/Art/hero.mat
  1 references    Assets/Art/hero.fbx
  0 references    Assets/Art/Textures/hero_BaseColor.png
```

---

## 跳过的资源

以下类型资源会被自动跳过，不会修改其 GUID：

- **场景文件**（`.unity`）— 场景内部引用结构复杂，跳过以防止破坏
- `.meta` 文件不存在或 GUID 不匹配的资源

跳过的资源会在结果对话框和 Console 中以红色标注。

---

## 注意事项

- **操作前务必备份或提交版本控制**，GUID 修改会直接写入磁盘，无法通过 Ctrl+Z 撤销
- 项目资源越多，第一阶段扫描耗时越长（万级资源约需数十秒）
- 不支持处理 `.unity` 场景文件，如需处理请手动操作
- 操作完成后 Unity 会自动刷新资源数据库，无需手动 Reimport
