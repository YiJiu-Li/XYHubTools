# Git 推送监听

监听 Git 远程仓库是否有新推送，并在 Unity Editor 内以**可拖动的非模态小窗**通知你。不会阻塞开发，也不会自动拉取或修改本地文件。

> 默认关闭，需要在面板里主动开启监听才会自动 fetch。

---

## 适用场景

- 团队协作时，队友 push 后想第一时间知道，避免拉取冲突
- 长时间挂着 Unity 开发，不想频繁切到终端看 `git status`
- 多人共用一个项目分支，希望自动感知远程更新

---

## 核心特性

- **非模态通知**：检测到新 commit 时在主编辑器窗口右上角弹出可拖动小窗
- **只读监听**：只执行 `git fetch` 和本地 refs 比对，不执行 pull / merge / rebase
- **可调间隔**：默认 60 秒一次 fetch，可在面板里改为 10-3600 秒
- **FileSystemWatcher 加速**：其它工具（如 VS Code）fetch 时能即时响应
- **推送历史**：面板内保留最近 50 次推送记录，可以逐条重新显示通知
- **可关闭推送合并**：多条推送会合并到通知历史，不刷屏
- **可自定义仓库路径**：默认跟随当前 Unity 项目根，也支持单独指定

---

## 快速开始

1. 打开 `XY Tools > 编辑器工具集`
2. 在左侧「项目工具」分类下找到 **Git 推送监听**
3. 右上角点击 **● 监听运行中** 按钮开启监听
4. 等待首次 fetch 完成（默认 3 秒后），面板会显示当前分支与远程状态
5. 当有人 push 到追踪的分支，右上角自动弹出通知

## 界面说明

### 状态区
- ✓ 与远程同步 — 本地与远程 HEAD 一致
- ↓ 落后远程 N 个 commit — 有新推送未拉取
- ↑ 领先远程 N 个 commit — 本地有未推送的 commit
- 显示当前分支、远程名、本地/远程 hash、上次 fetch 时间

### 控制区
- **● 监听运行中 / ○ 监听已停止** — 总开关
- **立即 fetch** — 手动执行 `git fetch origin --prune`
- **立即检查** — 不执行 fetch，只读取现有 ref 做比对

### 设置区
- **轮询间隔(秒)** — 范围 [10, 3600]，推荐 60
- **远程名** — 默认 `origin`
- **追踪分支** — 留空 = 跟随当前 HEAD；切换后会自动重置 last-seen 状态
- **自动 fetch** — 关闭后只能手动触发 fetch
- **通知方式** — 可拖动小窗 / 仅日志
- **自动消失(秒)** — 通知窗口自动隐藏，0 = 不自动消失
- **仓库路径** — 高级设置，留空 = 当前 Unity 项目根

### 推送历史区
- 保留最近 50 次推送
- 每条记录支持「重新显示通知」

## 安全边界

| 操作 | 是否自动执行 | 触发方式 |
|------|--------------|----------|
| `git fetch` | 是（可关闭） | 定时器 / 手动 |
| `FileSystemWatcher` | 是 | `.git/refs/` 或 `FETCH_HEAD` 变化时 |
| 显示通知 | 是 | 检测到新 commit |
| `git pull` | **否** | 本工具不提供 pull 操作 |

- 所有 git 命令使用 `Process`，`CreateNoWindow = true`，不会闪控制台
- 不在主线程同步执行，UI 不会卡顿
- 找不到 git 命令或仓库时会清晰报错，不会静默失败

---

## 已知限制

- 仅支持 Windows（依赖 `git` 命令在 PATH 中可用；macOS / Linux 也能跑，未测）
- 同一分支同一 commit 不会重复通知（last-seen 状态持久化）
- 不支持多远程同步监听，只追踪设置的单一远程
- 本工具只负责提示远端是否有推送；拉取、合并、解决冲突请在你习惯的 Git 工具中完成

---

## 故障排查

| 现象 | 可能原因 |
|------|----------|
| 状态显示「未找到 .git 目录」 | 项目根没有 `.git`，或路径覆盖填写错误 |
| 状态显示「— 未知」 | 远程分支不存在（如删除了远程分支） |
| 通知不弹 | 检查设置中的「通知方式」是否为「仅日志」 |
| fetch 超时 | 仓库大或网络慢，可调大轮询间隔 |
